Experiments

This page provides access to the experimental data generated for the evaluation of Anatlyzer reported in the paper Static Analysis of Model Transformations (submitted to IEEE TSE).

The generated data is available in a zip file. The following provides information about the files and how to reproduce the experiments. If you wisth to reproduce the experiments you have to first install anatlyzer, including the anATLyzer Experiments feature (or build anatlyzer from the sources). This is a facility that we have implemented to run automated experiments for ATL transformations.

Mutants

To analyse the accuracy of anatlyzer we have generated a set of mutants from an error-free transformation. The mutants have been executed against a large set of input models in order to try to produce runtime errors. The execution of each mutant has been compared against the output of anatlyzer is able to accurately report those runtime errors.

The results of the experiment are in two files (within experimental_data.zip):

  • evaluation_mutants.xlsx : Result of each mutant, plus a summary of the experiment
  • mutants.expdata : XML file with the raw data. It was used to produce the previous file.

To reproduce the experiment you have to import one of these two projects:

  • mutation_experiment_all.zip (150 MB, 1 GB uncompressed!) : Includes the original transformations plus all the generated mutants and test models. To execute the experiment right-click on transformation/factory2pn3.atl -> Evaluate anATLyzer -> Lite/Full evaluation. The evaluation may take several hours to complete.
    • .
  • mutation_experiment_only_trafo.zip : Includes only the original transformation. If the experiment is run with this project it will automatically generate a new set of mutants and models.

Analysis of the ATL Zoo

We have used anatlyzer to analyse the complete ATL Zoo. The results of the experiment are in the following files (within folder zoo_analysis in experimental_data.zip):
  • zoo_analysis_byError.xlsx : Summary of problems ocurrences, grid relating transformations and problems, and list of all problems.
  • zoo_analysis_byKind : Summary based on the kind of problems.
  • zoo_analysis_bySeverity : Summary based on the severity of problems.
  • zoo_analysis_only_solver : Summary of the behaviour of the model finder.
  • zoo_analysis.expdata : XML file with the raw data. It was used to produce the previous files.

To execute the experiment you have to import the anatlyzer.evaluation.atlzoo project. It contains a file named zoo_analysis.exp with the configuration of the experiment. Open it, go to the Result tab and click on Execute. One caveat is that you need to have the ATL Zoo projects available in your workspace and properly configured (we provide here our snapshot, hopefully not violating any license). The experiment may take several ours to complete.

Performance benchmarks

The results of the performance benchmarks are in the following files (within folder performance in experimental_data.zip):

  • error-path.xlsx : Performance results using error-path strategy, that is, using the smallest input meta-model for the model finder.
  • mand-full.xlsx : Performance results using mandatory strategy, which sometimes produces larger input meta-models but generate fully conforming witness models
  • full.xlsx : Performance results without any pruning strategy

      • To reproduce the experiment you have to import the anatlyzer.evaluation.performance project. Run one of the .exp files as before.
