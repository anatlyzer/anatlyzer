Quick fixes

Support for quick fixes has been added to Anatlyzer. This page provides information about this and the detailed results of our experiments.

Quick fixes in the ATL editor

To enable quick fixes in the ATL editor, anatlyzer must be activated. Please, follow the installation instructions and the user guide.

Given a problem reported in the Problem View or in the Analysis View, the quick fix proposals can be generated in three ways.

  • From the editor, press CTRL+1 (altough there are some known bugs with this)
  • From the Problem view, select the problem and press CTRL+1. A dialog with the proposals will be shown.
  • From the Analysis view, select the problem, right-click and select Quickfix. A dialog with the proposals will be shown.

Experiment results

The completeness of our quickfixes has been evaluated by generating mutants of a transformation in order to create a wide range of problems. The usefulness has been evaluated by manually quick fixing a transformation written by a third-party. In both cases the amount of applicable quick fixes and their impact after their application, using the following automation script: 

Input: transformation T, list of problems Lp

Step 1: Confirm or discard problems using the model finder
        If discarded, remove problem from Lp
Step 2: Foreach P in Lp
           Retrieve the set of available quick fixes for P
           Foreach quick fix, Q
              Copy T into Tq
              Apply Q on Tq
              Run the static analyser on Tq, obtaining
                   the problems, Lpq, remaining after Q
              Confirm or discard problems from Lpq, as in step 1
              Compare Lp and Lpq

This script has been carried out using a custom-made facility to automatically process transformations and meta-models and export the results in textual form or in Excel format. The detailed results and the data to replicate the experiments are available in the following files.

  • Complete project (anatlyzer.evaluation.quickfixes)
  • Results for mutants
  • Results for CPL2SPL

The anatlyzer.evaluation.quickfixes project includes two .exp files to easily run the experiments. For example, open quickfixes_test.exp, select the Result tab, and press Execute. After a few minutes, the results will appear in the text editor. The experiment over the mutated transformations may take more than 30 minutes.

To run the experiments Eclipse must be configured to run with a heap space of at least 2 GB. You must change eclipse.ini adding -Xms1024m -Xmx2048m.
